Lazy loading


É usado para baixar as informações da página, somente quando for aberta, para não sobrecarregar o site.
Exemplo o site tem três páginas se abrir todas juntas, vai ser mais demorado,
se abrir somente a que você clicou para entrar, ela abrirá mais rápido.


Vamos criar um "módulo", que pode conter diversos componentes, vamos separar nossos códigos.

No pront de comando, verifique se está dentro da pasta do projeto, e coloque :
ng g module lazy-loading --route=lazy-loading --module=app.module, dê enter.

--route=lazy-loading => já passa a rota que será acessado esse módulo.
--module=app.module => já será atrelado ao "app.module.ts".

Depois de criado, abre a pasta dele, e veja que ele criou novo módulo lazy-loading.module.ts.

No app-routing.module.ts, veja a rota que ele criou é diferente das outras rotas,

path:'lazy-loading' o endereço da rota.
loadChildren: () que pede para fazermos uma função.
=> import('./lazy-loading/lazy-loading.module') pede para importa e passa o caminho .
then((m) => m.LazyLoadingModule) e vai carregar esse módulo.

path:'lazy-loading',loadChildren: () => import('./lazy-loading/lazy-loading.module').then((m) => m.LazyLoadingModule),

Ele também criou o "router", no lazy-loading-routing.module.ts do nosso módulo,
já definiu as rotas, no nosso caso só tem a rota inicial,
que vai acionar o nosso módulo mostrando o seu template, a sua página.

Vamos vê funcionando para entender a diferença.
Abra a página, clique em "F12",
ou clique sobre a página com o botão direito do mouse e escolha "Inspecionar",

vai no "Network", atualize a página e veja os arquivos que serão carregados.


Agora na barra de navegação digite:lazy-loading, e veja quais os arquivos foram carregados.

Esse arquivo só foi carregado, quando executamos a página "lazy-loading",
esse é o conceito do "lazy-loading", só carregar os arquivos se for precisar deles.
